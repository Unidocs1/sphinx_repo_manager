Platform Installation using Helm¶

This article details the steps to install the AcceleratXR platform to Kubernetes using Helm. Before you begin make sure that all pre-requisites <prerequisites> have been installed and configured correctly.

Clone the Helm Chart¶

A Helm chart capable of deploying the platform including all supported micro-services, databases and stats servers is available in GitLab. Select the URL of the repository for the subscription plan you have below.

Next clone the repository to your local machine.

1git clone git@gitlab.acceleratxr.com:Core/tools/k8s_deploy.git

Dependencies¶

Before the helm chart can be used the dependencies must be locally downloaded and initialized. This is done with the following command.

1helm dep up

Create a Namespace¶

It is recommended that AcceleratXR be installed within a dedicated namespace within Kubernetes.

1kubectl create axr-demo-v1

In the above example we create a namespace called axr-demo-v1. This namespace will contain all of the platform’s resources.

Utilizing In-Cluster Database Servers¶

The following command will install the AcceleratXR platform and set up all necessary database servers within the running cluster. This is the RECOMMENDED install method.

 1helm install axr-demo-v1 . \
 2--namespace axr-demo-v1 \
 3--set title=AXR-Demo \
 4--set domain=demo.goaxr.cloud \
 5--set ingress.hosts[0].host=api.demo.goaxr.cloud \
 6--set mongodb.create=true \
 7--set mongodb.architecture=standalone \
 8--set mongodb.auth.enabled=false \
 9--set mongodb.auth.username="admin" \
10--set mongodb.auth.password="<PASSWORD>" \
11--set mongodb.auth.rootPassword="<PASSWORD>" \
12--set postgresql.create=true \
13--set postgresql.postgresqlPassword="<PASSWORD>" \
14--set admin.password="<PASSWORD>"

Utilizing External Database Servers¶

If external database providers are desired, such as using DocumentDB/RDS/Elasticache when running in AWS, the following command should be used.

 1helm install axr-demo-v1 . \
 2--namespace axr-demo-v1 \
 3--set title=AXR-Demo \
 4--set domain=demo.goaxr.cloud \
 5--set ingress.hosts[0].host=api.demo.goaxr.cloud \
 6--set mongodb.create=false \
 7--set mongodb.url=mongodb://admin:<PASSWORD>@ext.hosted.mongodb \
 8--set mongodb.auth.username="admin" \
 9--set mongodb.auth.password="<PASSWORD>" \
10--set mongodb.auth.rootPassword="<PASSWORD>" \
11--set postgresql.create=false \
12--set postgresql.url=postgres://admin:<PASSWORD>@ext.hosted.postgresql \
13--set postgresql.postgresqlPassword="<PASSWORD>" \
14--set admin.password="<PASSWORD>"
